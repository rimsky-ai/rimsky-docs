# rimsky-docs

Public documentation, lint tooling, and compilable reference examples for
[rimsky](https://github.com/fallguyconsulting/rimsky), the reactive
node-graph orchestration platform.

## Layout

- `docs/` — public documentation (49 markdown files). Concepts,
  protocols, agents guide, blob-backend / store / executor references,
  patterns, and the operator guide.
- `cmd/` — Go tooling (own `go.mod`, stdlib + `gopkg.in/yaml.v3` only).
  Three binaries:
  - `rimsky-docs-lint` — four structural lints (frontmatter, glossary
    parity, citation drift, and `llms.txt` validity). Subcommand `all`
    runs every lint. These check mechanical correctness only; word
    choice and user-facing clarity are the doc-writing agents' judgment.
  - `rimsky-docs-llms-full` — concatenates the rimsky concept bodies and
    `docs/protocols/*.md` into `docs/agents/llms-full.txt` plus a
    repo-root `llms-full.txt` copy.
  - `rimsky-docs-glossary` — publishes `docs/glossary.md` as a verbatim
    copy of rimsky's concept catalog (`.ok-planner/design/concepts.md`).
- `examples/` — compilable Go reference impls (own `go.mod`, imports
  `github.com/fallguyconsulting/rimsky/protocols`). Today contains
  `atomic-staging-fs-producer/` with four scenario tests under
  `scenarios/`.
- `llms.txt`, `llms-full.txt` — repo-root copies of the agent-oriented
  retrieval files (verbatim mirrors of `docs/agents/llms.txt` and
  `docs/agents/llms-full.txt`; regenerated by `rimsky-docs-llms-full`).

## The `RIMSKY_REPO` convention

Every binary under `cmd/` requires the `RIMSKY_REPO` environment
variable to point at a local rimsky checkout. The binaries exit
non-zero with a helpful message when it is unset. The convention is
load-bearing for the pre-release reconciliation gate: rimsky's release
script invokes the lint tooling from a pinned `rimsky-docs` checkout
with `RIMSKY_REPO` set to the about-to-release rimsky tip, so drift
between docs and source surfaces before any tag is cut.

Typical invocation from a local rimsky-docs checkout sitting next to a
local rimsky checkout:

```bash
cd cmd && RIMSKY_REPO=$(pwd)/../../rimsky go run ./rimsky-docs-lint all
cd cmd && RIMSKY_REPO=$(pwd)/../../rimsky go run ./rimsky-docs-llms-full
cd cmd && RIMSKY_REPO=$(pwd)/../../rimsky go run ./rimsky-docs-glossary
```

Bypass for emergency rimsky releases lives at
`rimsky/scripts/release.sh` behind `--skip-docs-reconciliation`.

## Building and testing

```bash
cd cmd && go build ./... && go test ./...
cd examples && go build ./... && go test ./...
```

Both modules use local `replace` directives in `examples/go.mod` so a
sibling rimsky checkout resolves without a published tag; CI strips or
pins those directives when validating against a tagged rimsky release.
