# Rimsky for humans

A five-page tour for an engineer deciding whether to engage with rimsky
for an agentic-workflow problem. It frames what rimsky is, the mental
model and vocabulary you need, the things that will surprise you, a
worked example end to end, and where to go next.

This is the human-readable companion to the project's
[`README`](https://github.com/fallguyconsulting/rimsky/blob/main/README.md).
If you are a coding agent, start instead at
[`../agents/llms.txt`](../agents/llms.txt).

1. [What rimsky is](01-what-rimsky-is.md) — the one-paragraph claim, who
   it is for, and what it was built against.
2. [Mental model and vocabulary](02-mental-model.md) — the nouns you
   need before anything else makes sense.
3. [The surprises](03-surprises.md) — what a reader pattern-matching
   against other orchestrators will not expect, including loops and
   recursion.
4. [A worked example, end to end](04-worked-example.md) — a held
   subgraph that stages work and commits or rolls back atomically.
5. [Where to go next](05-where-to-go-next.md) — handoff to the agent
   docs, the concept catalog, the protocol guides, and the cookbook.

Read them in order on a first pass. They are short by design; depth
lives in the [concept catalog](../glossary.md) and the
[protocol guides](../protocols/README.md).
